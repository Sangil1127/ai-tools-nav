# AI工具导航网站 - 版本管理规范

## 存档机制（必须严格执行）

### 每次更新前必须执行
```bash
# 1. 运行更新前检查
./scripts/pre-update-check.sh

# 2. 确认工具数量正确（81个）
# 3. 确认所有元素完整
# 4. 检查通过后才能更新
```

### 每次更新后必须执行
```bash
# 1. 运行版本存档
./scripts/version-archive.sh

# 2. 确认Git提交成功
# 3. 确认备份文件生成
# 4. 确认版本日志更新
```

### 关键检查点

| 检查项 | 标准 | 失败处理 |
|--------|------|----------|
| 工具数量 | ≥81个 | 停止更新，从备份恢复 |
| 官方Logo | 全部有favicon | 补充缺失的logo |
| 热度系统 | 有热度条 | 修复热度显示 |
| 搜索功能 | 可用 | 修复搜索代码 |
| 筛选功能 | 可用 | 修复筛选代码 |
| Git同步 | 本地=远程 | 强制推送 |

### 版本号规则
```
vYYYY.MM.DD-HHMM
例如: v2026.03.15-1949
```

### 备份文件位置
```
backups/
├── backup-20260315_194900.tar.gz
├── backup-20260315_195000.tar.gz
└── ... (保留最近10个)
```

### 版本日志位置
```
VERSION_LOG.md
```

### 紧急恢复流程

**如果发现数据丢失：**

1. **立即停止更新**
   ```bash
   git status  # 确认状态
   ```

2. **从备份恢复**
   ```bash
   # 找到最新备份
   ls -t backups/*.tar.gz | head -1
   
   # 解压恢复
   tar -xzf backups/backup-XXXX.tar.gz
   ```

3. **从Git历史恢复**
   ```bash
   # 找到上次正确的提交
   git log --oneline | head -10
   
   # 回滚到正确版本
   git reset --hard <commit_hash>
   git push --force-with-lease
   ```

4. **验证恢复结果**
   ```bash
   ./scripts/pre-update-check.sh
   ```

### 责任制度

- **每次更新必须验证**
- **数据丢失必须追责**
- **必须确保81个工具完整**
- **版本存档必须成功**

### 禁止事项

- ❌ 未经检查直接提交
- ❌ 工具数量不匹配时强行更新
- ❌ 不备份直接修改
- ❌ 忽略验证脚本警告

### 脚本说明

| 脚本 | 用途 | 执行时机 |
|------|------|----------|
| pre-update-check.sh | 更新前检查 | 每次修改前 |
| version-archive.sh | 版本存档 | 每次提交后 |
